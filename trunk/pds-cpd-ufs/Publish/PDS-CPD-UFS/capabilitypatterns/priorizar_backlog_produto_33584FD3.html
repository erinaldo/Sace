Task: Priorizar Backlog do Produto
O Responsável pelo produto prioriza periodicamente o Backlog od produto.
Purpose

Garantir que a Equipe de Desenvolvimento está construindo o produto correto.

Main Description

O Responsável pelo Produto deve indicar a importância para cada história identificada. Deve-se informar uma pontuação entre 0 e 100. Mais pontos siginifica maior importância. É recomendado inicialmente deixar espaços entre as histórias, exemplo, 0-20-40-60-80-100.

Quando fichas manuais forem utilizadas deve-se solicitar essa informação e anotá-la nas fichas. Essa pontuação deverá ser transferida e/ou atualizada na planilha que contém o Backlog do Produto para definir a prioridade das funcionalidades que serão desenvolvidas.

A planilha criada/atualizada deverá ser colocada no Nuxeo já que ela deverá ser acessada ou atualizada por qualquer membro da Equipe Scrum.
