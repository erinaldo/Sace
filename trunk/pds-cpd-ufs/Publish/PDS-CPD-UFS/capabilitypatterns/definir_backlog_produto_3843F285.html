Task: Criar Backlog do Produto
Identifica as funcionalidades do produto que será desenvolvido.
Purpose

Identificar as funcionalidades do produtos

Main Description
Identifica as funcionalidades do produto que será desenvolvido através de reuniões com os usuários.
Steps
Entrevistar Usuário

Deve-se marcar reuniões com o Responsável pelo Produto e através de entrevistas entender qual o problema a ser resolvido, anotando todas as necessidades do usuário.

O tempo para cada reunião não deve ultrapassar 2 horas.

Ao final da reunião o Scrum Master do Produto deve criar o Backlog do Produto.


Descrever Histórias

Após cada entrevista o Scrum Master do Produto deve descrever as Histórias identificadas. O modelo para descrição dessas histórias pode ser acessado através Site de Desenvolvimento do CPD.

Para facilitar a conversa com o Responsável pelo Produto é recomendável a utilização de fichas manuais para descrição das histórias. No Site de Desenvolvimento do CPD existe um exemplo de como essas fichas seriam utilizadas. Cada história seria descrita em uma ficha. Quando os requisitos estiverem estabilizados o Scrum Master do Produto deverá trascrever as histórias identificadas e armazená-las no Nuxeo para futura aprovação pelo usuário.

Cada história deve conter:

  • Número Identificador Único.
  • Nome
  • Descrição
  • Como demonstrar que a funcionalidade funciona.

As histórias devem ser escritas de forma simples para que possam ser entendidas por toda Equipe Scrum.
