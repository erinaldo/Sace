Visão Geral do Processo de Desenvolvimento de Software do CPD-UFS
Main Description

O processo de Desenvolvimento de Software do CPD-UFS tem como objetivo padronizar todo o processo necessário para o desenvolvimento de software de alta qualidade no CPD da Universidade Federal de Sergipe.

O processo é fortemente baseado no Scrum.  O Scrum é um processo ágil de desenvolvimento de software no qual o progresso do projeto é controlado através de uma série de sprints que possuem duração fixa. No CPD o tempo padrão de duração de um Sprint são 3 semanas.

No início de cada sprint uma Reunião de Planejamento do Sprint é realizada durante o qual o Responsável pelo Produto prioriza o Backlog do Produto. A Equipe de Desenvolvimento de Sistemas seleciona então as atividades que poderão executar no sprint. As tarefas selecionadas farão parte do Backlog do Sprint.

Durante a execução do sprint devem ser realizadas diariamente reuniões diárias (15 minutos em pé) com o objetivo de ajudar a equipe a pernacer na trilha correta. A reunião diária não é para solução de problemas e ajuda a evitar reuniões desnecessárias. Todos integrantes são convidados a responder três peguntas essenciais:

  • O que fizeste ontem?
  • O que vais fazer hoje?
  • Há algum obstáculo?

A respostas não são um relatório, mas um compromisso entre os pares. O scrum master representa a gerência do projeto, sendo o principal responsável por aplicar os valores e práticas do scrum.

No fim de cada sprint a equipe deve demonstrar a funcionalidade completa do Sprint a todo Equipe Scrum.
